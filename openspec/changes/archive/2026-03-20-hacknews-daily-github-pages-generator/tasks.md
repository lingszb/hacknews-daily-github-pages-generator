## 1. 数据采集与输入边界

- [x] 明确日报生成的输入范围，定义“当日候选 stories”的来源边界与采集时间窗口。
- [x] 定义候选 story 的最小输入字段集合，确保后续标准化具备明确前提。
- [x] 明确输入中需要保留的原始信息类型，例如原始标题、源链接、story 类型、可用排序信号。
- [x] 约定输入批次与发布日期的关系，确保一次生成对应一个明确的 digest date。
- [x] 明确采集失败、部分采集成功、空结果三类输入状态在系统中的处理语义。

## 2. 统一内容模型

- [x] 起草统一 story 内容模型，覆盖原始数据、标准化字段、筛选状态、栏目归属、展示字段、可选中文摘要字段。
- [x] 起草 digest 内容模型，覆盖发布日期、固定栏目集合、栏目内条目列表、归档标识、首页指向关系。
- [x] 明确哪些字段属于必填发布字段，哪些字段属于可选增强字段。
- [x] 明确 story 与 digest 的身份标识规则，确保同一日期 digest 可被唯一识别。
- [x] 明确模型中的失败与排除状态表达，以支持后续验证与降级。

## 3. 标准化与筛选

- [x] 定义候选 story 标准化规则，确保不同输入项可转换为统一记录。
- [x] 明确“标题保持原始标题”的规则，并作为发布前验证条件之一。
- [x] 定义 digest eligibility 规则，明确哪些 story 可进入后续编排候选池。
- [x] 定义单条异常 story 的排除规则，确保单项异常不会导致整批失败。
- [x] 定义“最小可发布候选集合”的判断标准，用于后续失败降级。

## 4. 日报栏目编排

- [x] 明确固定栏目集合：今日头条、热门讨论、值得深读、Ask HN、Show HN。
- [x] 定义栏目分类规则，使 curated stories 能被归入固定栏目。
- [x] 明确 Ask HN 与 Show HN 的分类优先级与识别条件。
- [x] 明确栏目内排序的判定原则，保证同一输入下输出稳定。
- [x] 定义栏目稀疏时的行为，确保栏目数量不足时仍可生成日报。
- [x] 定义“今日头条”“值得深读”与其他栏目的展示差异边界，为可选中文摘要预留位置。

## 5. 日报页面输出

- [x] 明确日报静态页面的最小输出结构，包括发布日期、栏目标题、story 列表、源链接。
- [x] 明确首页即最新日报的输出约束，保证首页不作为独立内容面。
- [x] 明确日报页面与首页之间的一致性校验规则。
- [x] 明确页面输出中不得出现中文标题替换原始标题的约束。
- [x] 明确可选增强字段缺失时的页面降级展示规则。

## 6. 历史归档发布

- [x] 明确历史 digest 的归档组织方式，保证每个发布日期都有稳定历史入口。
- [x] 明确新日报发布时对历史归档的追加行为，不覆盖既有历史内容。
- [x] 明确归档列表最少应展示的识别信息，例如日期与访问入口。
- [x] 明确首页切换到新日报时，旧日报仍保留为历史页的规则。
- [x] 明确当次新构建失败时，历史归档保持不变的要求。

## 7. GitHub Actions 定时发布

- [x] 明确 GitHub Actions 作为每日调度与发布机制的职责边界。
- [x] 定义构建流程阶段：采集、标准化、筛选、分类、页面生成、发布。
- [x] 明确定时触发与手动重跑均应支持的发布语义。
- [x] 明确发布成功的判定条件，以及发布失败时不得破坏既有已发布站点的要求。
- [x] 明确构建日志与失败定位所需的阶段性输出信息，便于后续排查。

## 8. 可选中文摘要支持

- [x] 明确中文摘要为可选字段，而非日报生成前置条件。
- [x] 明确仅“今日头条”和“值得深读”允许显示一句中文摘要/推荐理由。
- [x] 明确“热门讨论”“Ask HN”“Show HN”不得显示中文摘要。
- [x] 明确中文摘要为附加说明，不得替代原始标题。
- [x] 明确每条 entry 最多一条中文摘要，避免多行说明破坏页面一致性。
- [x] 明确缺失中文摘要时的输出行为，确保页面仍然有效。

## 9. 失败降级与鲁棒性

- [x] 明确单条 story 标准化失败时的跳过策略。
- [x] 明确部分栏目缺项时的可发布策略。
- [x] 明确可选中文摘要缺失时的继续发布策略。
- [x] 明确新一轮生成失败时首页与历史归档的保护策略。
- [x] 明确“何时允许降级发布、何时必须整体失败”的判定边界。

## 10. 基础验证与验收

- [x] 为 6 个 capability 分别建立可验证的验收检查点，覆盖输入、输出与约束。
- [x] 验证同一输入下栏目分类与输出结构的一致性。
- [x] 验证首页始终对应最新一期已发布 digest。
- [x] 验证历史 digest 在多日连续发布后仍可访问。
- [x] 验证原始标题保留规则始终成立。
- [x] 验证中文摘要仅出现在允许栏目，且为可选单行附加信息。
- [x] 验证部分失败条件下系统仍可输出最小可发布日报。
- [x] 验证无法形成最小可发布日报时，本次发布失败且不破坏既有站点。

- [x] 再落地 homepage 与 archive publishing 规则。
- [x] 最后落地 selective chinese summary support 这一可选增强能力。
